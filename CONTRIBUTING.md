## How to Contribute

### 1. Fork the repository
Create your own fork of the project and clone it locally.

### 2. Create a new branch
Use a clear branch name, e.g.: feature/calendar-improvements fix/task-edit-bug

### 3. Make your changes
Keep code clean, readable, and consistent with the existing style.  
StudyHub Web uses React, JavaScript/JSX, and local storage for data.

### 4. Test your changes
Make sure the app builds and runs without errors: 
```bash
npm install 
```
then
```bash
npm run dev
```


### 5. Submit a pull request
Open a PR to the `main` branch.  
Include a short description of what you changed and why.

## Code Style
- Use clear, descriptive variable and component names  
- Keep components small and focused  
- Prefer functional components and hooks  
- Avoid unnecessary dependencies  

## Issues & Suggestions
If you find a bug or want to propose a feature, open an issue with:
- A clear title  
- Steps to reproduce (if applicable)  
- Screenshots or examples (optional)
